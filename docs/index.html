Política de Privacidade

1. Introdução

Esta Política descreve como tratamos informações no portal. Nosso compromisso é com a transparência técnica e a segurança, operando sob uma arquitetura de dados que prioriza a privacidade por padrão (Privacy by Design).

2. Coleta de Dados

Nossa coleta de dados é estritamente funcional e limitada ao necessário para a operação:

  • Comunicação: Não coletamos endereços de e-mail. Toda a comunicação e suporte são realizados exclusivamente através de meios internos de mensagem do portal.
  • Dados Fiscais e Pagamento: Dados como CPF, CNPJ e informações de meios de pagamento são coletados apenas para viabilizar transações financeiras e a emissão de notas fiscais, quando necessário.
  • Logs Técnicos: Endereços IP e metadados de conexão são registrados para segurança e auditoria interna.

3. Distinção entre Dados Reais e Virtuais

Atenção: Há uma separação técnica e jurídica entre dados externos (vida real) e dados internos da conta dentro da simulação criada pelo portal.

Neste "Mundo Virtual", os dados referentes a personagens, atributos de jogo, conquistas e interações simuladas não estão sujeitos à LGPD, pois não constituem dados pessoais identificáveis fora do contexto da simulação.

Aconselhamos fortemente que os usuários não utilizem dados reais para compor nomes de caracteres ou interações dentro do mundo virtual.

4. Cookies e Tecnologias

O portal utiliza apenas cookies de sessão. Diferente de outras plataformas, armazenamos dados analíticos e essenciais diretamente em bancos de dados no servidor, minimizando o rastreamento no lado do cliente (Client-side).

Reiteramos: dados gerados no âmbito do mundo virtual não são sujeitos à LGPD, sendo tratados como metadados da simulação.

5. Armazenamento e Criptografia

Todos os dados sensíveis necessários são armazenados exclusivamente sob formas computacionalmente seguras:

  • Utilizamos HASH, HMAC HASH e chaves de criptografia avançadas.
  • Os dados são inacessíveis até mesmo para a equipe de desenvolvimento.

Como não armazenamos dados em formato literal, não possuímos meios técnicos para recuperar senhas ou chaves perdidas. A perda dessas credenciais implica na perda irremediável do acesso aos dados protegidos.

6. Segurança de API

A segurança das nossas APIs é gerenciada pelo servidor utilizando chaves independentes, segregadas da conta de usuário do desenvolvedor.

// Os dados trafegam e são validados via HMAC
const securityHeader = {
    'X-HMAC-Signature': '<HASH_DE_INTEGRIDADE>',
    'X-Server-Key': '<INTERNAL_SERVER_ONLY>'
};

Novamente, todo dado coletado via API que necessite de persistência é armazenado em formato de HASH ou criptografia, mantendo a política de não-recuperação de dados literais e proteção total contra vazamentos de dados legíveis.
